# Phase 3 – Topic 1: Why Loops?

Before learning the `for`, `while`, and `do...while` loops, you need to understand **why loops exist**.

This topic is mostly conceptual, but it lays the foundation for everything that follows.

---

# 1. What is a Loop?

A **loop** is a programming construct that allows you to **repeat a block of code multiple times** without writing the same code again.

Instead of writing the same statement repeatedly, you write it **once** and let the loop execute it as many times as needed.

---

# 2. Why Do We Need Loops?

Imagine you want to print:

```text
Hello
Hello
Hello
Hello
Hello
```

Without loops:

```js
console.log("Hello");
console.log("Hello");
console.log("Hello");
console.log("Hello");
console.log("Hello");
```

Works ✔️

But what if you need to print it **1,000 times**?

You'd have to write:

```js
console.log("Hello");
console.log("Hello");
console.log("Hello");
// ...
// 997 more lines
```

That's inefficient.

---

## Using a Loop

```js
for (let i = 1; i <= 5; i++) {
    console.log("Hello");
}
```

Output

```text
Hello
Hello
Hello
Hello
Hello
```

One small block of code replaces five repeated statements.

---

# 3. Real-Life Analogy

Imagine you're a teacher taking attendance.

Without a loop:

```text
Is Roll No. 1 present?
Is Roll No. 2 present?
Is Roll No. 3 present?
...
```

With a loop:

```text
Repeat:
Ask the next student until everyone is checked.
```

A loop automates repetitive work.

---

# 4. Where Are Loops Used?

Loops are everywhere in programming.

### Arrays

```js
const fruits = ["Apple", "Banana", "Mango"];
```

Instead of:

```js
console.log(fruits[0]);
console.log(fruits[1]);
console.log(fruits[2]);
```

Use a loop.

---

### Database Records

Imagine MongoDB returns:

```js
[
    { name: "John" },
    { name: "Alice" },
    { name: "Bob" }
]
```

You use a loop to display every user.

---

### Shopping Cart

```text
Calculate total price

Item 1
Item 2
Item 3
Item 4
```

A loop processes every item automatically.

---

### React

Rendering a list of products:

```text
Laptop

Phone

Tablet

Mouse
```

Internally, React often uses iteration to render each item.

---

# 5. What Happens Inside a Loop?

Every loop follows the same basic cycle.

```text
Start

↓

Check condition

↓

True?

↓

Run code

↓

Update

↓

Go back to condition

↓

False?

↓

Stop
```

This cycle repeats until the condition becomes false.

---

# 6. Types of Loops in JavaScript

## `for`

Best when you know how many times to repeat.

Example:

```text
Print numbers 1–100
```

---

## `while`

Best when the number of repetitions is unknown.

Example:

```text
Keep asking for a password until it's correct.
```

---

## `do...while`

Runs at least once, even if the condition is false.

Example:

```text
Show a menu before checking whether to exit.
```

---

## `for...of`

Used to iterate over arrays, strings, maps, and other iterable objects.

---

## `for...in`

Used to iterate over the keys (property names) of an object.

---

# 7. Common Mistakes

## Mistake 1 – Avoiding Loops

❌

```js
console.log(1);
console.log(2);
console.log(3);
console.log(4);
console.log(5);
```

✔️

```js
for (let i = 1; i <= 5; i++) {
    console.log(i);
}
```

---

## Mistake 2 – Choosing the Wrong Loop

Use:

* `for` → Known number of iterations.
* `while` → Unknown number of iterations.
* `for...of` → Arrays and strings.
* `for...in` → Objects.

---

# 8. Best Practices

✅ Use meaningful variable names.

```js
for (let index = 0; index < 10; index++) {
}
```

instead of

```js
for (let x = 0; x < 10; x++) {
}
```

when the purpose is indexing.

---

✅ Keep loop bodies simple.

---

✅ Avoid repeating code when a loop can do the job.
