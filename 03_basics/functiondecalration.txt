# Phase 6 – Topic 1: Function Declaration

---

# 1. What is a Function?

A **function** is a reusable block of code that performs a specific task.

Instead of writing the same code repeatedly, you write it once inside a function and call it whenever needed.

---

# 2. Why Do We Need Functions?

Without functions:

```js
console.log("Welcome");
console.log("Welcome");
console.log("Welcome");
console.log("Welcome");
```

This repeats the same code.

Using a function:

```js
function welcome() {
    console.log("Welcome");
}

welcome();
welcome();
welcome();
welcome();
```

The code is shorter, easier to maintain, and reusable.

---

# 3. Real-Life Analogy

Think of a **coffee machine**.

You press the "Coffee" button.

```
Button Press

↓

Machine Executes Steps

↓

Coffee Ready
```

Similarly,

```js
login();
```

means:

```
Call login()

↓

JavaScript runs the login function

↓

Login completed
```

# 6. Calling (Invoking) a Function

Creating a function does **not** execute it.

```js
function greet() {
    console.log("Hello");
}
```

Nothing happens until you call it.

```js
greet();
```

Output

```text
Hello
```

---

# 7. Internal Working

```js
function greet() {
    console.log("Hello");
}

greet();
```

Execution Flow

```
JavaScript reads function

↓

Stores it in memory

↓

Does NOT execute

↓

Finds greet()

↓

Jumps to function body

↓

Runs code

↓

Returns to next line
```

---

# 8. Calling Multiple Times

```js
function greet() {
    console.log("Hello");
}

greet();
greet();
greet();
```

Output

```text

Hello
Hello
```

One function can be reused as many times as needed.

# 11. Function Naming Rules

### Good

```js
function login() {}
function calculateTotal() {}
function printName() {}
function getUserData() {}
```

---

### Bad

```js
function a() {}
function x() {}
function data() {}
```

Choose names that describe what the function does.

---

# 12. Function Naming Convention

Use **camelCase**.

```js
calculateArea()
printMarks()
fetchUserData()
sendEmail()
```

# 13. Common Mistakes

## ❌ Forgetting Parentheses

```js
function greet() {
    console.log("Hello");
}

greet;
```

Output

Nothing happens because the function is **referenced**, not called.

Correct

```js
greet();
```

---

## ❌ Calling Before Creating? (Special Case)

```js
greet();

function greet() {
    console.log("Hello");
}
```

Output

```text
Hello
```

This works because **function declarations are hoisted**.

We'll study hoisting in Phase 7.

---

## ❌ Expecting a Function to Run Automatically

```js
function greet() {
    console.log("Hello");
}
```

Output

Nothing.

You must call it.

```js
greet();
```

# 14. Best Practices

* ✅ Give functions meaningful names.
* ✅ One function should do one job.
* ✅ Keep functions short and readable.
* ✅ Reuse functions instead of duplicating code.

# 18. Interview Questions

### Q1. What is a function?

**Answer:** A reusable block of code that performs a specific task.

---

### Q2. What is a function declaration?

**Answer:** A function created using the `function` keyword.

```js
function greet() {}
```

---

### Q3. What is the difference between declaring and calling a function?

* **Declaring:** Defining the function.
* **Calling:** Executing the function.

---

### Q4. Can a function be called multiple times?

**Answer:** Yes. A function can be reused as many times as needed.

---

### Q5. Are function declarations hoisted?

**Answer:** Yes. Function declarations are hoisted, so they can be called before their definition.
